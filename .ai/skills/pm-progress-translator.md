# PM Progress Translator (The Value Bridge)

> [!IMPORTANT] **G-Stack Methodology**: Every translation begins with
> **Tech-Stack Discovery**. The agent must understand the "Contextual Impact" of
> a technical change before framing it. Follow **MinimumCD** by celebrating the
> completion of small, verifiable logic units.

## 🎯 Verification Gates (Clarity & Impact)

### Phase 0: Tech-Stack Discovery (MANDATORY)

- **Action**: identify the recent changes via `git log` or `AnalyticsEvent`
  history.
- **Goal**: Differentiate between "Maintenance Churn" and "Feature Development."

### Gate 1: The "So What?" Filter

- **Positive (Signal):** successfully linked a low-level change (e.g., "Updated
  JWT logic") to a high-level value (e.g., "Enhanced user security and login
  reliability").
- **Negative (Noise):** Listing file names or package updates as "Status."
- **Action**: If Negative, pivot to "System Health" framing.

### Gate 2: Stakeholder Empathy

- **Positive (Pass):** The report uses non-technical language optimized for
  decision-makers.
- **Negative (FAIL):** Using terminology like "hydration," "middleware," or
  "normalization" without explanation.

## 📋 Outcome Actions

- **Deliver**: A "Stakeholder Status Report" formatted for emails or slack
  briefings.
- **Ethos**: Translation is Transformation. Change "I fixed X" into "The system
  is now faster/safer/better for users."
